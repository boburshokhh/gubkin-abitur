# Документация по базе данных

В этой директории содержится документация по структуре и функциям базы данных системы приема абитуриентов.

## Содержание

- [Схема базы данных](../database-schema.md) - полное описание таблиц, полей и связей
- [Функции базы данных](database-functions.md) - документация по функциям PostgreSQL с примерами использования

## Основные компоненты базы данных

### Справочники
- `education_levels` - уровни образования (бакалавриат, специалитет)
- `directions` - направления подготовки
- `subjects` - предметы для вступительных испытаний
- `document_types` - типы документов
- `application_statuses` - статусы заявлений

### Основные таблицы
- `users` - пользователи системы
- `profiles` - профили образовательных программ
- `applications` - заявления абитуриентов
- `application_choices` - выбранные программы в заявлении (до 3-х)
- `documents` - загруженные документы

### Вспомогательные таблицы
- `profile_exams` - связь профилей и экзаменов
- `application_history` - история изменений статусов заявлений
- `admin_logs` - журнал действий администраторов

## Ключевые функции

### Для абитуриентов
- `get_profiles()` - получение списка доступных профилей
- `create_application()` - создание заявления
- `get_compatible_profiles()` - получение совместимых профилей для выбора

### Для администраторов
- `get_applications()` - получение списка заявлений с фильтрацией
- `update_application_status()` - изменение статуса заявления
- `get_direction_statistics()` - получение статистики по направлениям

## Безопасность

Безопасность реализована с помощью:
- Row-Level Security (RLS) политик для таблиц
- Проверок прав доступа в функциях (`is_admin()`, `is_admin_or_reviewer()`)
- Логирования действий пользователей и администраторов
